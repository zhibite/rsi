# RSI 马丁策略 — 核心策略说明

> 文档版本：2026-03-09
> 适用项目：`d:/05.project2/1.bquant/rsi`

---

## 策略概述

本系统采用 **RSI 超卖信号入场 + 马丁格尔分批补仓 + 多出场机制** 的现货量化策略。

核心思路：
1. 利用 RSI 指标判断价格短期超卖，在相对低位建仓
2. 若价格继续下跌，按预设跌幅分批加仓（马丁补仓），降低持仓均价
3. 价格反弹后通过多种止盈机制出场，锁定利润

所有逻辑均以**现货买卖**实现，无杠杆，最大亏损为持仓本金（不会爆仓）。

---

## 一、RSI 超卖入场

### 指标计算

使用 **Wilder RSI（14 周期，1 分钟 K 线）**，通过 OKX WebSocket 实时推送的行情数据计算。

```
RSI = 100 - 100 / (1 + RS)
RS  = 近 N 周期平均涨幅 / 近 N 周期平均跌幅   （Wilder 平滑法）
```

RSI 含义：
- RSI < 30：超卖区间，价格可能超跌，具有反弹潜力
- RSI > 70：超买区间，价格可能超涨，存在回调压力
- RSI ≈ 50：中性，多空均衡

### 入场条件

同时满足以下三个条件时，执行首单买入：

```
① RSI(14, 1m) < rsi_oversold（默认 30）
② 该交易对当前无开仓持仓
③ 当前总开仓数 < max_open_positions（默认 5）
```

### 订单执行

市价买入，金额为 `first_order_amount`（默认 20 USDT）。实际买入数量：

```
持仓数量 = 买入金额 / 成交价格 × (1 - 0.1%)   # 扣除手续费
```

买入后记录：`avg_price`（均价）、`total_invested`（总投入）、`total_quantity`（持仓量）

## 二、马丁格尔补仓

### 触发条件

持仓期间，若价格从**上次买入价格**下跌超过当前层对应的跌幅阈值，触发补仓：

```
跌幅 = (上次买入价 - 当前价) / 上次买入价 × 100
若 跌幅 >= grid_drops[当前层] → 执行下一层补仓
```

### 补仓金额

每层补仓金额按倍数递增（默认 2 倍）：

| 层级 | 补仓金额（默认） | 触发跌幅（默认）|
|------|----------------|----------------|
| L0（首单） | 20 U | — |
| L1（第 1 次补仓） | 40 U | 1% |
| L2（第 2 次补仓） | 80 U | 2% |
| L3（第 3 次补仓） | 160 U | 3% |
| L4（第 4 次补仓） | 320 U | 4% |
| L5（第 5 次补仓） | 640 U | 5% |
| **累计投入** | **1260 U** | |

> 补仓金额 = `first_order_amount` × `martin_multiplier` ^ 层级

### 均价更新

每次补仓后重新计算均价：

```
新均价 = (原总投入 + 本次买入金额) / (原总持仓量 + 本次买入量)
```

补仓不断摊低均价，使价格反弹到一个较低水平时就能盈利出场。

### 补仓回撤确认（可选）

若配置了 `replenishment_retracement_pct > 0`（默认 0.3%）：

```
价格跌破触发线 → 记录最低价（谷底）
价格从谷底反弹 >= replenishment_retracement_pct% → 才执行补仓
```

作用：避免在瀑布式下跌的中途补仓，等待价格出现反弹迹象后再买入。

## 三、止盈出场机制

系统提供**三种止盈方式**，按优先级依次检查：

```
优先级：硬止损 > RSI 超买立即止盈 > 追踪止盈（启动线 + 回撤）
```

### 3.1 RSI 超买立即止盈

当 RSI 进入超买区域且利润达标时，跳过追踪止盈，**直接立即卖出**。

#### 触发条件

同时满足：
```
① RSI >= effective_overbought（有效超买阈值）
② 净利润% >= overbought_min_profit（最低利润要求）
```

#### 梯度阈值设计

随马丁层级加深，RSI 阈值和最低利润要求同步降低，补仓越深越容易触发出场：

```
有效超买 RSI = max(50, rsi_overbought - overbought_rsi_step × 马丁层级)
最低利润%   = max(0.3%, overbought_min_profit_pct - overbought_profit_step × 马丁层级)
```

**示例（默认参数：RSI 基准 70，步长 2；利润基准 1%，步长 0.1%）：**

| 层级 | RSI 超买阈值 | 最低净利润 | 最低净利润（以 20U 首单计） |
|------|------------|-----------|--------------------------|
| L0（20U投入）   | 70.0 | 1.0% | 0.20 U |
| L1（60U投入）   | 68.0 | 0.9% | 0.54 U |
| L2（140U投入）  | 66.0 | 0.8% | 1.12 U |
| L3（300U投入）  | 64.0 | 0.7% | 2.10 U |
| L4（620U投入）  | 62.0 | 0.6% | 3.72 U |
| L5（1260U投入） | 60.0 | 0.5% | 6.30 U |

> 最低利润 0.3% 为绝对下限（覆盖双边手续费 0.2% + 余量）

#### 利润计算方式

使用**净利润**（已扣除双边手续费），非毛利：

```
净收益 = 持仓量 × 当前价格 × (1 - 0.1%)   # 扣卖出手续费
净利润% = (净收益 - 总投入) / 总投入 × 100
```

买入时已扣过买入手续费（体现在持仓量减少），故净利润已含双边手续费影响。

---

### 3.2 追踪止盈

适用于 RSI 未达超买、但持仓已有一定盈利的场景，通过"跟涨回撤"锁定更高利润。

#### 触发流程

```
阶段①：监控净利润
  若 净利润% >= take_profit_pct（默认 1.3%）
    → 激活追踪止盈，记录当前价格为 peak_price

阶段②：追踪价格高峰
  每个周期更新 peak_price = max(peak_price, 当前价格)

阶段③：回撤止盈
  若 (peak_price - 当前价格) / peak_price × 100 >= trailing_stop_pct（默认 0.3%）
    → 执行卖出，锁定利润
```

**示意：**
```
价格
  │         ▲ peak_price
  │        /│
  │       / │← trailing_stop_pct = 0.3%
  │      /  ▼ 回撤触发卖出
  │     /
  │────/──── take_profit_pct = 1.3% 启动线
  │   /
  └──────────── 时间
  ↑ 开仓均价
```

#### 参数含义

| 参数 | 含义 |
|------|------|
| `take_profit_pct` | 追踪止盈启动线，净利润达到此值后开始追踪（默认 1.3%） |
| `trailing_stop_pct` | 从最高价回撤此值时卖出（默认 0.3%） |

---

### 3.3 止盈对比

| 方式 | 触发条件 | 特点 |
|------|---------|------|
| RSI 超买立即止盈 | RSI 高位 + 利润达标 | 不等最高点，RSI 转向前出场，适合短线快出 |
| 追踪止盈 | 利润达启动线后价格回落 | 尽量追高，适合趋势行情 |

两者可**同时配置**，先满足条件的先触发。

## 四、扫描机制

### WebSocket 实时推送

系统使用 OKX WebSocket 订阅所有配置交易对的 **1 分钟 K 线**，实时更新：
- RSI 值（每根 K 线闭合后重算）
- 当前价格（Tick 级别实时推送）

数据保存在内存中（`okx_ws_scanner`），Bot 每个扫描周期直接读取，无需发送 REST API 请求。

### 扫描周期

Bot 每隔 `scan_interval`（默认 2 秒）执行一次扫描：

```
扫描周期内：
  1. 读取 BTC 价格 → 熔断检查
  2. 遍历所有开仓持仓 → 止盈/补仓/止损判断
  3. 遍历所有启用交易对（无仓）→ RSI 入场信号检查
  4. 休眠 scan_interval 秒
```

### 多用户隔离

每个用户独立运行一个 asyncio Task，拥有独立的：
- Bot 状态（运行/停止）
- API 密钥
- 持仓记录
- RSI 缓存
- 风控状态（熔断计时器等）

## 五、手续费说明

OKX 现货 Taker 手续费：**0.1%**（`FEE_RATE = 0.001`）

| 操作 | 手续费 |
|------|--------|
| 买入 | 买入金额 × 0.1%（体现为收到的代币数量减少） |
| 卖出 | 卖出金额 × 0.1%（体现为到账 USDT 减少） |
| 双边合计 | ~0.2% |

**盈亏平衡点**：价格需涨约 **0.2%** 才能覆盖双边手续费（严格计算约 0.2002%）。

建议最低止盈利润不低于 **0.3%**（覆盖手续费后还有约 0.1% 净收益）。

## 六、自动化保护

### 交易对自动禁用

若某交易对在交易所不可用（如仅支持实盘的合约在模拟盘不存在），下单失败后会：
1. 自动禁用该交易对（`is_enabled = False`）
2. 记录 WARNING 日志
3. 后续扫描周期不再尝试

### 手动操作互斥

通过仪表板手动买入/卖出时，Bot 会暂时跳过该交易对的自动管理，避免并发冲突。

### 余额不足兜底

若卖出时交易所返回"余额不足"错误：
1. 查询交易所实际持仓余额
2. 用实际余额重试卖出
3. 若余额为 0（持仓已在其他地方消耗），强制关闭数据库仓位记录

## 七、策略适用场景

### 适合

- **震荡市**：价格在一定范围内反复波动，RSI 超卖后反弹概率高
- **短期超跌**：突发消息导致价格急跌，RSI 进入超卖区后往往有快速修复
- **主流山寨币**：流动性好，点差小，不容易被操纵

### 不适合

- **单边下跌趋势**：马丁补仓会持续亏损，补满后仍无法回本
- **流动性极差的小币**：点差大，频繁交易手续费损耗高
- **极端黑天鹅事件**：价格断崖式下跌且长期不恢复（建议配合风控方案）
