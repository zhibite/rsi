# RSI 马丁策略 — 配置参数完整参考

> 文档版本：2026-03-09
> 配置路径：Web 界面 → 配置页面（`/settings.html`）

---

## 一、OKX API 配置

| 字段 | 说明 |
|------|------|
| 账户标签 | 自定义标识，仅用于显示 |
| API Key | OKX 创建的 API Key |
| Secret Key | 对应的 Secret |
| Passphrase | 创建 API 时设置的密码 |
| 模式 | **模拟盘**（推荐测试）/ **实盘**（真实资金）|

> API 密钥使用 Fernet 对称加密存储，主密钥保存在 `.env` 文件中，首次启动自动生成。

## 二、RSI 与扫描参数

| 参数 | 字段名 | 默认值 | 范围 | 说明 |
|------|--------|--------|------|------|
| RSI 超卖阈值 | `rsi_oversold` | `30` | 10~50 | RSI 低于此值触发入场 |
| RSI 周期 | `rsi_period` | `14` | 5~50 | Wilder RSI 计算周期（K 线根数）|
| 扫描间隔 | `scan_interval` | `2` | ≥1 | 每次扫描之间的休眠秒数 |
| 最大同时持仓数 | `max_open_positions` | `5` | 1~50 | 超过此数量不开新仓 |

---

## 三、超买立即止盈参数

RSI 进入超买区域且利润达标时，直接卖出（不走追踪止盈）。

| 参数 | 字段名 | 默认值 | 说明 |
|------|--------|--------|------|
| RSI 超买基准 (L0) | `rsi_overbought` | `70` | 首单（L0）持仓的超买 RSI 阈值 |
| 每层 RSI 步长 | `overbought_rsi_step` | `2` | 每深一个马丁层级，RSI 阈值降低的幅度 |
| 最低净利润基准 % (L0) | `overbought_min_profit_pct` | `1.0` | L0 要求的最低净利润（已含手续费）|
| 每层利润步长 | `overbought_profit_step` | `0.1` | 每深一层，最低利润要求降低的幅度 |

### 各层有效阈值计算公式

```
有效超买 RSI  = max(50,   rsi_overbought - overbought_rsi_step × 层级)
最低净利润%   = max(0.3%, overbought_min_profit_pct - overbought_profit_step × 层级)
```

### 默认参数下各层阈值

| 层级 | 有效 RSI 阈值 | 最低净利润 |
|------|-------------|-----------|
| L0 | 70.0 | 1.0% |
| L1 | 68.0 | 0.9% |
| L2 | 66.0 | 0.8% |
| L3 | 64.0 | 0.7% |
| L4 | 62.0 | 0.6% |
| L5 | 60.0 | 0.5% |

## 四、马丁格尔补仓参数（全局默认值）

以下参数为新增交易对时的默认值，可在**交易对管理**中单独覆盖。

| 参数 | 字段名 | 默认值 | 说明 |
|------|--------|--------|------|
| 首单金额 | `first_order_amount` | `20` USDT | 第一笔买入金额 |
| 马丁倍数 | `martin_multiplier` | `2.0` | 每层补仓金额 = 首单 × 倍数^层级 |
| 最大补仓次数 | `max_martin_levels` | `5` | 最多补仓 5 次（L1~L5）|
| 止盈启动线 | `take_profit_pct` | `1.3%` | 净利润达此值后激活追踪止盈 |
| 追踪止盈回撤 | `trailing_stop_pct` | `0.3%` | 从最高价回撤此值时卖出 |
| 补仓回撤确认 | `replenishment_retracement_pct` | `0.3%` | 价格跌破补仓线后，需反弹此值才执行补仓（0=立即补仓）|
| 补仓跌幅梯度 | `grid_drops` | `[1,2,3,4,5]%` | 各层补仓的触发跌幅，共 5 档 |

### 默认倍数下各层资金分配（首单 20U，倍率 2x）

| 层级 | 本次买入 | 累计总投入 |
|------|---------|-----------|
| L0（首单）| 20 U | 20 U |
| L1 | 40 U | 60 U |
| L2 | 80 U | 140 U |
| L3 | 160 U | 300 U |
| L4 | 320 U | 620 U |
| L5 | 640 U | 1260 U |

## 五、风险控制参数

详见 [risk-control.md](risk-control.md)，此处为速查表。

| 参数 | 字段名 | 默认值 | 说明 |
|------|--------|--------|------|
| 硬止损浮亏 % | `max_loss_pct` | `null`（关闭）| 持仓净浮亏超此值强制平仓 |
| 补仓冷却时间 | `martin_cooldown_seconds` | `0`（不限）| 两次补仓之间最短间隔秒数 |
| BTC 熔断跌幅 % | `btc_drop_pct` | `5.0` | BTC 短时跌幅触发熔断阈值 |
| BTC 熔断监测窗口 | `btc_drop_minutes` | `15` 分钟 | 监测 BTC 跌幅的时间窗口 |
| 熔断后暂停开仓 | `btc_pause_minutes` | `60` 分钟 | 熔断触发后暂停新开仓时长（0=关闭熔断）|

## 六、交易对管理

每个交易对可独立配置以下参数（覆盖全局默认值）：

| 参数 | 字段名 | 说明 |
|------|--------|------|
| 交易对 | `symbol` | 格式：`BTC/USDT`（OKX 现货）|
| 启用状态 | `is_enabled` | 关闭后 Bot 不扫描该交易对 |
| 首单金额 | `first_order_amount` | 覆盖全局默认值 |
| 马丁倍数 | `martin_multiplier` | 覆盖全局默认值 |
| 最大补仓次数 | `max_martin_levels` | 覆盖全局默认值 |
| 止盈启动线 | `take_profit_pct` | 覆盖全局默认值 |
| 追踪止盈回撤 | `trailing_stop_pct` | 覆盖全局默认值 |
| 补仓回撤确认 | `replenishment_retracement_pct` | 覆盖全局默认值 |
| 补仓跌幅梯度 | `grid_drops` | 覆盖全局默认值 |
| 代币精度 | `lot_sz` | OKX 最小下单单位，点击「同步代币精度」自动获取 |

### 批量获取 TOP 50

点击交易对管理页面中的「批量获取 TOP 50」按钮，系统自动：
1. 从 OKX 获取现货交易量前 50 的 USDT 交易对
2. 跳过已配置的交易对
3. 使用当前全局默认参数批量创建

## 七、参数调整建议

### 保守配置（低风险低收益）

```
首单金额：10 U
倍率：1.5x
最大补仓层：4
RSI 超卖：28
补仓跌幅：[1.5, 3, 5, 8]%
止盈启动线：1.0%
```

### 均衡配置（推荐）

```
首单金额：20 U
倍率：2x
最大补仓层：5
RSI 超卖：30
补仓跌幅：[1, 2, 3, 4, 5]%
止盈启动线：1.3%
追踪回撤：0.3%
补仓回撤：0.3%
超买 RSI：70，步长 2
最低利润：1.0%，步长 0.1%
```

### 激进配置（高风险高收益）

```
首单金额：50 U
倍率：2x
最大补仓层：5
RSI 超卖：32（更频繁入场）
补仓跌幅：[0.8, 1.5, 2.5, 4, 6]%（更密集补仓）
止盈启动线：0.8%（更快止盈）
```

## 八、参数影响速查

| 想要… | 调整参数 | 方向 |
|-------|---------|------|
| 更频繁开仓 | `rsi_oversold` | 调高（如 32~35） |
| 更少开仓 | `rsi_oversold` | 调低（如 25~28） |
| 降低均价更快 | `martin_multiplier` | 调高（如 2.5x） |
| 控制总资金暴露 | `max_martin_levels` 或 `first_order_amount` | 减小 |
| 更快止盈 | `take_profit_pct` | 调低 |
| 追求更高利润 | `take_profit_pct` | 调高，`trailing_stop_pct` 调低 |
| RSI 超买更灵敏 | `rsi_overbought` | 调低（如 65） |
| 防止瀑布踩踏 | `martin_cooldown_seconds` | 设置 180~300 |
| 防极端暴跌 | `max_loss_pct` | 设置 15~20 |
| 防市场恐慌 | `btc_drop_pct` + `btc_pause_minutes` | 开启熔断 |
