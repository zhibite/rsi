# RSI 马丁策略 — 风险控制方案

> 文档版本：2026-03-09
> 适用项目：`d:/05.project2/1.bquant/rsi`

---

## 背景

RSI + 马丁格尔策略在正常震荡行情中表现稳健，但面对**极端快速大跌**（如黑天鹅、流动性危机、交易所事件）时，马丁补仓会持续消耗资金，若价格长期不反弹则面临大幅亏损甚至爆仓风险。

本文档描述系统已实现的三层风控机制及其配置方法。

---

## 整体执行优先级

每个扫描周期内，持仓管理按以下顺序执行，高优先级触发后直接返回，跳过后续检查：

```
① 硬止损检查        ← 最高优先级，浮亏超阈值立即平仓
② RSI 超买立即止盈   ← 梯度阈值，RSI 达标 + 利润达标才卖
③ 追踪止盈          ← 启动线触发后跟踪回撤卖出
④ 马丁补仓          ← 冷却期内跳过；BTC 熔断期间跳过新开仓
```

---

## 一、硬止损（Hard Stop Loss）

### 原理

当持仓**净浮亏**（已扣卖出手续费）超过配置阈值时，不论 RSI 状态，强制市价全平仓位。

```
净浮亏% = (总投入 - 当前市值×(1-手续费率)) / 总投入 × 100
若 净浮亏% >= max_loss_pct → 立即平仓，记录 reason=stop_loss
```

### 配置参数

| 参数 | 字段名 | 默认值 | 说明 |
|------|--------|--------|------|
| 硬止损浮亏 % | `max_loss_pct` | `null`（关闭） | 填 0 或留空表示不启用 |

> 配置路径：**配置页面 → ⚠️ 风险控制参数 → 硬止损浮亏 %**

### 亏损额参考（首单 20U，倍率 2x）

| 止损设置 | L0（20U投入） | L1（60U投入） | L2（140U投入） | L3（300U投入） | L4（620U投入） | L5（1260U投入） |
|---------|--------------|--------------|----------------|----------------|----------------|----------------|
| 10% | 2 U | 6 U | 14 U | 30 U | 62 U | 126 U |
| 15% | 3 U | 9 U | 21 U | 45 U | 93 U | 189 U |
| 20% | 4 U | 12 U | 28 U | 60 U | 124 U | 252 U |

> 止损触发时的最大亏损 = **当前累计投入 × max_loss_pct%**

### 建议值

- 保守：`10%`（亏损较小，但容易被正常波动触发）
- 推荐：`15%`（兼顾保护与容忍度）
- 激进：`20%`（适合资金充裕、补仓层数多的场景）

### 注意事项

- 设置过小会在正常回调中频繁止损，之后价格反弹时已出场
- 止损是**确定亏损**，只有在认为价格不会反弹时才有意义
- 建议配合补仓冷却一起使用，避免在瀑布急跌中快速补满再触发止损

## 二、补仓冷却（Martin Cooldown）

### 原理

两次马丁补仓之间强制等待最短时间，防止价格连续快速下跌时在短时间内触发多个补仓层。

```
本次补仓时间 - 上次补仓时间 < martin_cooldown_seconds → 跳过本次补仓
达到冷却时间后 → 以届时价格执行下一层补仓
```

### 配置参数

| 参数 | 字段名 | 默认值 | 说明 |
|------|--------|--------|------|
| 补仓冷却时间 | `martin_cooldown_seconds` | `0`（不限） | 单位：秒 |

> 配置路径：**配置页面 → ⚠️ 风险控制参数 → 补仓冷却时间 (秒)**

### 效果示例（设置 300 秒冷却）

```
10:00:00  价格跌破 L1 触发线 → 执行 L1 补仓，记录时间
10:01:30  价格继续跌破 L2 触发线 → 距上次 90s < 300s → 跳过
10:02:00  价格继续跌破 L3 触发线 → 距上次 120s < 300s → 跳过
10:05:10  价格仍在 L3 线以下 → 距上次 310s >= 300s → 执行 L2 补仓

瀑布急跌期间：只触发了 1 次补仓，而非连续 3 次
```

### 权衡

| | 无冷却 | 有冷却（300s） |
|--|--------|--------------|
| 急跌场景 | 快速消耗全部补仓层 | 只消耗部分补仓层，剩余资金保留 |
| 缓慢下跌场景 | 正常分批补仓 | 正常分批补仓（影响不大） |
| 补仓价格 | 尽量接近触发价 | 冷却结束时可能价格更低（均价更低） |

### 建议值

- 平稳行情：`0`（不设冷却）
- 高波动市场：`180`~`300` 秒（3~5 分钟）
- 极端保守：`600` 秒（10 分钟）

## 三、BTC 联动熔断（Circuit Breaker）

### 原理

监控 BTC/USDT 价格短时跌幅，当超过阈值时判断为系统性市场恐慌，**暂停所有新开仓**，已有持仓继续正常管理。

```
每个扫描周期：
  记录 BTC 当前价格到内存缓冲（最多保留 2000 条）

  若熔断已激活且未到期 → circuit_open = True → 跳过新开仓

  若熔断未激活：
    取过去 btc_drop_minutes 分钟内最早价格 P0 和当前价格 P1
    跌幅 = (P0 - P1) / P0 × 100
    若跌幅 >= btc_drop_pct → 触发熔断
      → 记录日志（WARNING 级别）
      → 设置熔断到期时间 = 当前时间 + btc_pause_minutes 分钟
      → circuit_open = True → 本周期跳过新开仓

熔断到期后自动恢复，下个周期重新检查
```

### 配置参数

| 参数 | 字段名 | 默认值 | 说明 |
|------|--------|--------|------|
| BTC 熔断跌幅 % | `btc_drop_pct` | `5.0` | 0 = 关闭熔断功能 |
| 熔断监测窗口 | `btc_drop_minutes` | `15` | 单位：分钟 |
| 熔断后暂停开仓 | `btc_pause_minutes` | `60` | 单位：分钟，0 = 关闭熔断 |

> 配置路径：**配置页面 → ⚠️ 风险控制参数 → BTC 熔断相关参数**

> ⚠️ `btc_pause_minutes = 0` 时整个熔断功能关闭，即使配置了跌幅阈值也不会触发。

### 效果示例（默认：5% / 15 分钟 / 暂停 60 分钟）

```
14:00  BTC = 80,000 U
14:15  BTC = 75,800 U → 跌幅 = 5.25% >= 5%

→ ⚡ 熔断触发，日志：
  "BTC熔断触发: 15min内跌幅>=5%, 暂停开新仓60min"

14:15 ~ 15:15
  · ETH/USDT RSI = 25 → 超卖信号出现 → 忽略，不开新仓
  · 已有 SOL/USDT 持仓 → 继续正常追踪止盈 / 马丁补仓
  · BTC/USDT 价格继续记录，等待到期

15:15  熔断到期自动解除 → 恢复正常开仓
```

### 为什么已有持仓不受熔断影响？

1. 已开仓的亏损已经存在，强制平仓反而锁定亏损
2. 已有持仓的马丁补仓是在更低位**降低均价**，逻辑上合理
3. 熔断核心目的是：**不在恐慌时开新坑**，而非清仓逃顶

### 数据来源

Bot 启动时强制订阅 `BTC/USDT` WebSocket 实时价格，即使用户未配置 BTC 交易对也会订阅，约每秒更新一次。BTC 价格历史保存在内存中（服务重启后清空，需约 `btc_drop_minutes` 分钟重新积累数据）。

## 三层风控协同示意

```
极端急跌场景：

 价格
  │
  │ ← 开仓（RSI 超卖）
  │
  ↓ 跌 5%  → BTC 熔断触发 → 停止开新仓
  ↓
  ↓ 跌 1%  → L1 补仓冷却中（300s内）→ 跳过
  ↓
  ↓ 跌 2%  → 冷却结束 → L1 补仓执行
  ↓
  ↓ 继续跌 → 冷却中 → 跳过 L2
  ↓
  ↓ 浮亏达 15% → 硬止损触发 → 强制全平

  结果：
  - 只执行了 1 次马丁补仓（而非全部层级）
  - 最终以 15% 净亏损出场
  - 资金损失可控，剩余资金可用于下次机会
```

## 推荐配置组合

### 保守型（适合资金较少、风险偏低）

| 参数 | 值 |
|------|----|
| 硬止损浮亏 % | `10` |
| 补仓冷却时间 | `300` 秒 |
| BTC 熔断跌幅 % | `3` |
| 熔断监测窗口 | `10` 分钟 |
| 熔断暂停开仓 | `120` 分钟 |

### 均衡型（推荐默认）

| 参数 | 值 |
|------|----|
| 硬止损浮亏 % | `15` |
| 补仓冷却时间 | `180` 秒 |
| BTC 熔断跌幅 % | `5` |
| 熔断监测窗口 | `15` 分钟 |
| 熔断暂停开仓 | `60` 分钟 |

### 激进型（适合资金充裕、追求收益）

| 参数 | 值 |
|------|----|
| 硬止损浮亏 % | `20`（或关闭） |
| 补仓冷却时间 | `0`（不限） |
| BTC 熔断跌幅 % | `8` |
| 熔断监测窗口 | `30` 分钟 |
| 熔断暂停开仓 | `30` 分钟 |
